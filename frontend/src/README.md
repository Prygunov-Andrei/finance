# Финансовое приложение для инженерной компании

Фронтенд-приложение для управления финансами, проектами, договорами и платежами.

## Быстрый старт

### 1. Настройка подключения к бэкенду

Создайте файл `.env.local` в корне проекта:

```bash
# Для локальной разработки
VITE_API_URL=http://localhost:8000/api/v1

# Или для удаленного сервера
# VITE_API_URL=https://your-server.com/api/v1
```

### 2. Запуск приложения

После настройки `.env.local` перезапустите приложение.

### 3. Авторизация

Используйте учетные данные Django для входа в систему.

## Возможные проблемы

### Ошибка "Failed to fetch"

Если вы видите ошибку подключения:

1. **Проверьте URL** в файле `.env.local`
2. **Убедитесь, что Django сервер запущен**
3. **Перезапустите приложение** после изменения .env

### Ngrok URL не работает

Ngrok URLs временные и могут устареть. Получите новый URL:

```bash
# На сервере с Django
ngrok http 8000
```

Затем обновите `VITE_API_URL` в `.env.local`

### CORS ошибки

Убедитесь, что ваш домен добавлен в:
- `ALLOWED_HOSTS` 
- `CSRF_TRUSTED_ORIGINS` 
- `CORS_ALLOWED_ORIGINS`

в настройках Django.

## Структура проекта

```
/
├── components/          # React компоненты
│   ├── catalog/        # Каталог товаров
│   ├── contracts/      # Договоры
│   ├── estimates/      # Сметы и проекты
│   ├── pricelists/     # Прайс-листы
│   ├── proposals/      # ТКП и МП
│   └── ui/             # UI компоненты
├── lib/                # API клиент и утилиты
├── types/              # TypeScript типы
└── styles/             # Глобальные стили
```

## Основные функции

✅ Dashboard с аналитикой  
✅ Управление платежами  
✅ Система договоров  
✅ Проекты и сметы  
✅ Прайс-листы с экспортом в Excel  
✅ Каталог товаров с модерацией  
✅ Парсинг PDF-счетов (в разработке)
