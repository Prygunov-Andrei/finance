# Инструкции: Коммерческие предложения

## Общий бизнес-процесс

Бизнес-процесс коммерческих предложений охватывает путь от первого контакта с потенциальным клиентом до подготовки полного пакета документов КП. Процесс проходит через два связанных канбана:

1. **Канбан поиска объектов** (раздел «Маркетинг») — начальный этап: поиск клиентов, назначение встреч, передача объекта на расчёт.
2. **Канбан КП** (раздел «Коммерческие предложения») — расчётный этап: подготовка смет, запрос счетов у поставщиков, утверждение и формирование КП.

### Механизм «Тоннель»

Оба канбана работают на одной доске `commercial_pipeline` с 12 колонками. Каждый канбан показывает только свои колонки:

| Маркетинг (6 колонок) | КП (6 колонок) |
|---|---|
| Новые клиенты | Новый расчёт |
| Назначена встреча | В работе |
| Проведена встреча | Счета запрошены |
| Расчёт подготовлен | Утверждение сметы |
| Нет результата | Смета утверждена |
| Есть результат | Подготовлено КП |

Когда карточка перемещается из **«Проведена встреча»** → **«Новый расчёт»**, она «исчезает» из канбана Маркетинга и появляется в канбане КП (первый столбец).

Когда расчёт готов и карточка перемещается в **«Подготовлено КП»** → **«Расчёт подготовлен»**, она возвращается обратно в канбан Маркетинга со всеми накопленными данными.

## Работа с канбаном КП

### Карточка канбана

Каждая карточка содержит:

- **Номер КП** — заголовок
- **Наименование объекта** — из справочника объектов
- **Система** — текстовое описание (например, «Вентиляция», «Кондиционирование»)
- **Потенциальный заказчик** — контактные данные: наименование юрлица, контактные лица, способы связи
- **Счета поставщиков** — файлы (PDF, Excel), полученные от поставщиков в процессе расчёта
- **Комментарии** — оперативные заметки
- **Привязанные ТКП** — ссылки на технико-коммерческие предложения

### Перемещение карточек

Используйте выпадающий список внизу каждой карточки для перемещения между колонками. Карточку можно перемещать в любую колонку всей доски (не только видимых).

## Создание ТКП (Технико-Коммерческое Предложение)

### Шаг 1: Основные поля

1. Откройте раздел **Коммерческие предложения → ТКП**
2. Нажмите **«Создать ТКП»**
3. Заполните обязательные поля:
   - **Название** — краткое описание ТКП
   - **Дата** — дата создания
   - **Объект** — выберите из справочника
   - **Компания** — ваше юрлицо

### Шаг 2: Привязка смет

1. Перейдите на вкладку **«Сметы»**
2. После выбора объекта отобразятся все сметы, созданные по этому объекту
3. Отметьте галочками нужные сметы (можно выбрать несколько)
4. Данные из смет будут автоматически скопированы в ТКП при сохранении

### Шаг 3: Фронт работ

1. Перейдите на вкладку **«Фронт работ»**
2. Пункты, отмеченные как «по умолчанию» в справочнике, уже выбраны
3. Вы можете:
   - Снять галочку с ненужных пунктов
   - Добавить дополнительные пункты
   - Указать сроки (текстом или конкретной датой)

### Шаг 4: Сохранение

Нажмите **«Создать»**. Система автоматически:
- Сгенерирует номер ТКП
- Привяжет выбранные сметы
- Создаст записи фронта работ

## Создание МП (Монтажное Предложение)

### Из списка МП

1. Откройте **Коммерческие предложения → МП**
2. Нажмите **«Создать МП»**
3. На вкладке «Основные поля»:
   - Выберите **объект**
   - Выберите **контрагента** (исполнителя)
   - При необходимости укажите **связанное ТКП**

### Выбор монтажных смет

1. Перейдите на вкладку **«Монтажные сметы»**
2. Выберите одну или несколько монтажных смет для данного объекта
3. Суммы и человеко-часы из смет будут учтены в МП

### Условия для МП

1. Перейдите на вкладку **«Условия для МП»**
2. Условия «по умолчанию» уже выбраны
3. Вы можете убрать ненужные или добавить дополнительные

### Из ТКП

Вы можете создать МП напрямую из карточки ТКП:
1. Откройте детальную страницу ТКП
2. Нажмите **«Создать МП из ТКП»**
3. Система автоматически заполнит объект и привязку к ТКП

## Справочники

### Фронт работ и монтажные условия

Справочники «Фронт работ» и «Условия для МП» находятся в разделе **Справочники и Настройки → Фронт работ и монтажные условия**.

Каждый элемент справочника имеет:
- **Название** — описание пункта
- **Активен** — включён/выключен
- **По умолчанию** — автоматически выбирается при создании ТКП/МП

### Потенциальные заказчики

Новый тип контрагента **«Потенциальный Заказчик»** используется для хранения контактных данных клиентов, с которыми ещё не заключён договор.

Создать потенциального заказчика можно:
- В разделе **Справочники → Контрагенты** (выбрать тип «Потенциальный Заказчик»)
- При создании карточки в канбане поиска объектов

## Роли и права доступа

| Роль | Доступ |
|---|---|
| Менеджер по продажам | Канбан КП, создание ТКП/МП, работа со сметами |
| Маркетолог | Канбан поиска объектов, создание потенциальных заказчиков |
| Снабженец | Прикрепление счетов поставщиков к карточкам канбана КП |
| Руководитель | Полный доступ ко всем разделам |

## Часто задаваемые вопросы

### Как привязать несколько смет к одному ТКП?

На вкладке «Сметы» в форме создания ТКП вы можете отметить любое количество смет для выбранного объекта.

### Как работает «тоннель» между канбанами?

Это один борд с 12 колонками. Канбан маркетинга показывает 6 колонок, канбан КП — другие 6. Перемещая карточку между «границами» видимости, вы передаёте её между отделами.

### Можно ли вернуть карточку из канбана КП в канбан Маркетинга?

Да. Используйте выпадающий список перемещения — все 12 колонок доступны. Переместите карточку в любую колонку маркетинга.
