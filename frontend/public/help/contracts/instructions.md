# Договоры — Инструкция пользователя

## Обзор раздела

Раздел **Договоры** — один из ключевых разделов ERP-системы. Здесь управляются три основные сущности:

- **Сметы** — источник истины по товарам и работам
- **Договоры и Допсоглашения** — юридические документы с условиями
- **Акты** — документы о выполненных работах (КС-2, КС-3)

Раздел связан с модулями **Финансы** (платежи, счета), **Каталог товаров** (продукция), **Прайс-листы** (расценки на работы), **Снабжение** (закупки).

## Навигация

### Боковое меню

В боковом меню раздел **Договоры** содержит:

| Пункт | Описание |
|-------|----------|
| Рамочные Договора | Управление рамочными договорами с Заказчиками и Исполнителями |
| Договора по объектам | Все договоры с фильтрами по объекту, типу, статусу |
| Сметы | Проектные сметы (этап ТКП/МП) |
| Монтажные сметы | Сметы для Исполнителей |
| Акты | Акты выполненных работ |
| Инструкции | Эта страница |

### Карточка объекта

В карточке каждого объекта есть вкладки:

- **Заказчик** → Сметы, Договоры и ДОП, Акты, Сверки
- **Исполнители** → Монтажные сметы, Договоры и ДОП, Акты, Сверки

## Сметы

### Что такое смета

Смета — это таблица с детальным перечнем товаров и работ, разбитая по разделам (системам). Каждая строка содержит:

- Номер по порядку
- Наименование товара / работы
- Модель / артикул
- Единица измерения
- Количество
- Цена материала за единицу
- Цена работы за единицу

### Жизненный цикл сметы

```
Спецификация → Проектная смета (ТКП) → Смета к Договору → Накопительная смета → Акты КС-2/КС-3
```

1. **Создание.** Сметчик загружает спецификацию (PDF/Excel). Система распознаёт и создаёт каркас сметы.
2. **Осмечивание.** Сметчик подбирает товары из каталога, запрашивает цены у поставщиков, подбирает расценки на работы.
3. **Согласование.** Смета согласуется с Заказчиком (или Исполнителем).
4. **Привязка к Договору.** Смета копируется как приложение к Договору. Может быть разбита на несколько монтажных смет.
5. **Закупки по смете.** Каждая закупка сопоставляется со строкой сметы — формируется накопительная смета.
6. **Акты.** Из накопительной сметы формируются КС-2 и КС-3.

### Автоматический подбор

При работе со сметой система автоматически:

- **Подбирает товар** из каталога по наименованию (fuzzy-matching + LLM для сложных случаев)
- **Подбирает цену** из истории закупок (последний счёт поставщика)
- **Подбирает работу** из прайс-листа (по истории + LLM для новых товаров)

Сметчик проверяет результат и при необходимости корректирует вручную.

### Аналоги

Если товар из спецификации недоступен, сметчик подбирает **аналог** с обязательным **обоснованием**. Эти обоснования сохраняются и используются при формировании допсоглашений.

### Накопительная смета

Накопительная смета — это подписанная смета, в которой для каждой строки отслеживается:

- Сколько закуплено, в каком количестве
- У какого поставщика, по какому счёту
- Есть ли отклонения (аналоги, превышение цены/количества)

Доступна в карточке объекта → Заказчик → Сметы, а также через API.

### Остатки по смете

Остатки = Смета - Накопительная. Показывают, что ещё не закуплено.

## Договоры

### Типы договоров

| Тип | Описание |
|-----|----------|
| **Доходный** (income) | Договор с Заказчиком |
| **Расходный** (expense) | Договор с Исполнителем |

### Статусы

| Статус | Описание |
|--------|----------|
| Планируется | Договор в процессе подготовки |
| Согласован | Текст согласован, ожидает подписания |
| В работе | Подписан, ведётся работа |
| Завершён | Все обязательства выполнены |
| Приостановлен | Работы временно остановлены |
| Расторгнут | Договор расторгнут |

### Рамочные договоры

Рамочный договор — договор, определяющий общие условия работы с контрагентом. На каждый объект создаётся Допсоглашение к рамочному договору.

### Допсоглашения (ДОП)

Допсоглашения вносят изменения в Договор:

- Изменение стоимости (новая смета)
- Изменение сроков
- ДОП к ДОП — цепочка изменений

### Текст договора

Подписанный текст договора хранится в формате Markdown для последующего поиска и анализа с помощью LLM.

## Акты

### Типы актов

| Тип | Описание |
|-----|----------|
| **КС-2** | Акт о приёмке выполненных работ (строго регламентированная форма) |
| **КС-3** | Справка о стоимости выполненных работ |
| **Простой акт** | Текстовый акт для физлиц или небольших работ |

### Статусы актов

| Статус | Описание |
|--------|----------|
| Черновик | Формируется, не отправлен Заказчику |
| Согласован | Заказчик ознакомлен и согласен |
| Подписан | Акт подписан обеими сторонами |
| Отменен | Акт отменён |

### Формирование КС-2

1. Откройте карточку объекта → Заказчик → Акты
2. Нажмите «Сформировать КС-2 из накопительной сметы»
3. Выберите строки из накопительной сметы
4. Укажите период работ и номер акта
5. Система сформирует акт с детальными строками

### Дебиторская задолженность

Из подписанных актов формируется дебиторская задолженность:

**Задолженность = Сумма подписанных актов - Сумма оплат**

Отображается во вкладке **Сверки** в карточке объекта.

## Контроль закупок по смете

### Проверка при оплате

При оцифровке счёта на оплату по Договору система автоматически:

1. Сопоставляет позиции счёта со строками сметы
2. Выявляет расхождения:
   - Превышение сметной цены
   - Превышение сметного количества
   - Замена товара на аналог
3. Требует обоснование для каждого расхождения

### Расходные материалы

Если строка сметы обобщённая (например, «Расходные материалы»), а в счёте — десятки позиций, формируется **субсмета** для этой строки.

### Дополнительные работы

Если товар или услуга вообще не предусмотрены сметой, формируется **субсмета «Допработы»** с обоснованием.

## Частые вопросы

**Как создать смету к договору?**
Смета создаётся при копировании проектной сметы (из ТКП) в Договорной отдел. Используйте действие «Создать из сметы» в разделе смет к договору.

**Как разбить смету для нескольких Исполнителей?**
Используйте действие «Разбить по разделам» в контекстном меню сметы к договору.

**Как сформировать КС-2?**
Откройте накопительную смету, выберите строки, нажмите «Сформировать акт».

**Почему система не даёт оплатить счёт?**
Если позиции счёта не сопоставлены со сметой — необходимо провести сопоставление вручную и обосновать расхождения.

## Работа со строками сметы

### Табличный редактор

Вкладка **«Строки сметы»** в карточке сметы содержит полнофункциональный табличный редактор:

- **Inline-редактирование** — кликните на ячейку для редактирования (наименование, модель, количество, цены)
- **Добавление строк** — кнопка «+ Строка» открывает форму
- **Вставка из Excel** — скопируйте строки из Excel (Ctrl+C), нажмите «Вставить из Excel» и вставьте (Ctrl+V) в текстовое поле
- **Массовое удаление** — отметьте строки чекбоксами и нажмите «Удалить»
- **Поиск** — используйте поле поиска для фильтрации по наименованию
- **Итоги** — внизу таблицы отображаются суммы по материалам, работам и общая сумма
- Строки-аналоги подсвечиваются жёлтым цветом

### Импорт из файла

Кнопка **«Импорт файла»** позволяет загрузить смету из Excel (.xlsx) или PDF:

1. Перетащите файл в зону загрузки или нажмите для выбора
2. Система автоматически распознает структуру (заголовки, разделы, строки)
3. Просмотрите результат распознавания и исправьте ошибки
4. Нажмите «Импортировать» для добавления строк в смету

**Excel** — обрабатывается локально, мгновенно. **PDF** — обрабатывается через LLM, может занять до минуты.

### Автоподбор цен и работ

Кнопка **«Подобрать цены»** запускает автоматический подбор:

1. Выберите прайс-лист (необязательно)
2. Система найдёт совпадения в каталоге товаров и прайс-листе работ
3. Для каждой строки показывается уровень уверенности (процент)
4. Вы можете принять или отклонить каждое совпадение
5. Кнопка «Принять все >80%» принимает все совпадения с высокой уверенностью

## Текст договора

Вкладка **«Текст договора»** на странице деталей договора позволяет:

- Хранить текст подписанного договора в формате Markdown
- Вести историю версий текста
- Переключаться между версиями
- Редактировать с предпросмотром (разделённый экран: редактор / предпросмотр)
- Привязывать текст к конкретному ДОП-соглашению

## Смета к договору

Страница **«Смета к договору»** (`/contracts/estimates/:id`) показывает:

- Полную таблицу строк с итогами по материалам, работам и общей суммой
- Статус сметы (черновик → согласована → подписана)
- Действия: согласовать, подписать, создать новую версию, скачать файл

## Акты

### Создание акта

Кнопка **«Создать акт»** доступна в списке актов:

- **КС-2** — выберите подписанную смету к договору, отметьте строки из накопительной сметы
- **КС-3 (справка)** — формируется аналогично КС-2
- **Простой** — укажите описание работ и сумму

### Страница акта

На странице деталей акта (`/contracts/acts/:id`):

- Просмотр суммы, НДС, неоплаченного остатка
- Таблица позиций (для КС-2)
- Действия: согласовать (черновик → согласован), подписать (согласован → подписан)
