# Проекты ПТО: Руководство пользователя

## Создание проекта

1. Перейдите в раздел **ПТО > Проекты**
2. Нажмите **Создать проект**
3. Заполните обязательные поля:
   - **Шифр проекта** (например, ПР-2025-001)
   - **Дата**
   - **Название проекта**
   - **Стадия** (Проектная документация или Рабочая документация)
   - **Объект** — выберите из списка или создайте новый (см. ниже)

### Быстрое создание объекта

Если нужного объекта нет в списке:
1. Нажмите кнопку **+** справа от поля "Объект"
2. В открывшемся окне заполните:
   - **Название** (обязательно)
   - **Адрес** (обязательно)
   - **Описание** (опционально)
3. Нажмите **Создать** — объект будет создан со статусом "Планируемый" и автоматически выбран в форме

### Загрузка файлов

К проекту можно прикрепить несколько файлов разных типов:

1. Нажмите **Добавить файлы** или перетащите файлы в область загрузки
2. Для каждого файла укажите:
   - **Тип файла** (выберите из справочника: Полный проект, Графическая часть, Спецификация, Техника и др.)
   - **Название** (опционально — дополнительное описание)
3. Можно добавлять файлы по одному кнопкой **Ещё файл**
4. Для удаления файла из списка нажмите иконку корзины

Поддерживаемые форматы: PDF, ZIP, DWG, и другие.

## Управление файлами проекта

На странице проекта (вкладка **Информация**) отображается таблица файлов:

| Столбец | Описание |
|---------|----------|
| Тип | Тип документа (из справочника) |
| Файл | Оригинальное имя файла |
| Название | Пользовательское описание |
| Загрузил | Кто загрузил файл |
| Дата | Дата загрузки |

Доступные действия:
- **Скачать** — нажмите иконку скачивания
- **Удалить** — нажмите иконку корзины (необратимо)
- **Загрузить новый файл** — кнопка **Загрузить файл** над таблицей

## Настройка типов файлов

Справочник типов файлов настраивается в разделе **Настройки > Типы файлов проектов**.

По умолчанию доступны:
- Полный проект
- Графическая часть
- Спецификация
- Техника

### Добавление нового типа

1. Перейдите в **Настройки > Типы файлов проектов**
2. Нажмите **Добавить тип**
3. Заполните:
   - **Название** — отображаемое имя
   - **Код** — уникальный идентификатор (латиница, snake_case)
   - **Порядок сортировки** — для упорядочивания в списках
   - **Активен** — неактивные типы скрыты из выбора

### Редактирование и удаление

- **Редактировать** — через меню действий (три точки) в строке типа
- **Удалить** — возможно только если к типу не привязаны файлы

## Замечания к проекту

На странице проекта, вкладка **Замечания** — журнал замечаний с поддержкой Markdown-форматирования.

### Добавление замечания

1. Перейдите на вкладку **Замечания**
2. Нажмите **Добавить замечание**
3. В окне редактирования доступны две вкладки:
   - **Редактор** — текстовое поле с поддержкой Markdown
   - **Предпросмотр** — как будет выглядеть замечание после сохранения
4. Нажмите **Добавить**

### Markdown-форматирование

В тексте замечаний поддерживается синтаксис Markdown:

| Синтаксис | Результат |
|-----------|-----------|
| `**жирный**` | **жирный** |
| `*курсив*` | *курсив* |
| `# Заголовок` | Заголовок уровня 1 |
| `## Заголовок` | Заголовок уровня 2 |
| `- пункт` | Маркированный список |
| `1. пункт` | Нумерованный список |
| `` `код` `` | Inline-код |
| `| A | B |` | Таблица (GFM) |

### Редактирование и удаление

- Автор замечания может **редактировать** и **удалять** свои замечания через иконки в правом углу карточки
- Чужие замечания доступны только для чтения

### Экспорт замечаний

Кнопка **Скачать .md** (рядом с «Добавить замечание») скачивает все замечания к проекту одним Markdown-файлом. Файл содержит заголовок проекта, а каждое замечание оформлено как раздел с автором и датой.

Кнопка неактивна, если замечаний нет.

## Перенос строк между разделами

Строки сметы можно переносить между разделами:

1. Выделите строку (чекбокс слева)
2. В поле вверху укажите номер целевой позиции
3. Нажмите **Перенести**
4. Строка переместится на указанную позицию и автоматически попадёт в соответствующий раздел

## Настройка ширины столбцов

Перетащите границу столбца в таблице строк сметы для изменения ширины. Ширина сохраняется автоматически — при возвращении на страницу сметы настройки ширины будут восстановлены.

## Файлы проектов в сметах

При создании сметы к ней привязываются проекты. Все файлы этих проектов отображаются на вкладке **Информация** сметы:

- Каждый проект показан с шифром и названием
- Под каждым проектом — список всех прикреплённых файлов с типом и ссылкой на скачивание

### Импорт строк из спецификации проекта

При импорте строк сметы (вкладка **Строки сметы** → **Импорт из файла**), помимо загрузки файла с диска, можно выбрать файл из проекта:

1. Откройте смету → вкладка **Строки сметы**
2. Нажмите **Импорт из файла**
3. В нижней части диалога появится секция **Файлы проекта**
   - Отображаются файлы с типом **Спецификация** из всех связанных проектов
   - Поддерживаются форматы: Excel (.xlsx) и PDF
4. Отметьте чекбоксами один или несколько файлов и нажмите **Импортировать выбранные**

**Для Excel-файлов:** откроется предпросмотр через несколько секунд.

**Для PDF-файлов:** запустится постраничная ИИ-обработка (как при прямом импорте PDF):
   - Отобразится прогресс-бар с номером текущей страницы
   - Строки будут появляться в таблице по мере распознавания
   - Можно свернуть диалог и продолжить работу
   - Обработка большого PDF может занять **10 и более минут**

5. Проверьте распознанные строки, при необходимости отметьте разделы
6. Нажмите **Импортировать** для добавления строк в смету

> **Совет:** Чтобы файл был доступен для импорта в смету, убедитесь что:
> - Файл загружен в проект с типом «Спецификация»
> - Проект привязан к смете (при создании или редактировании сметы)
> - Файл имеет формат Excel (.xlsx) или PDF
