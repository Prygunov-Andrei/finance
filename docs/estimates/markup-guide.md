# Система наценок в сметах

## Общий принцип

Сметчик работает с **закупочными (входными) ценами** — это цены, по которым мы покупаем товары у поставщиков и оплачиваем работы исполнителям. Система автоматически применяет **наценку** и вычисляет **продажные цены** для заказчика.

**Формула**: `Продажная цена = Закупочная цена + Наценка`

## Три уровня наценок

Наценки задаются на трёх уровнях. Более детальный уровень перекрывает более общий:

| Уровень | Где задаётся | Приоритет |
|---------|-------------|-----------|
| **Смета** | Вкладка "Информация" | Базовый (3-й) |
| **Раздел** | Вкладка "Разделы" | Средний (2-й) |
| **Строка** | Таблица строк / массовая наценка | Высший (1-й) |

### Пример каскада

- Смета: материалы 30%, работы 300%
- Раздел "Вентиляция": материалы 50% (работы наследуются — 300%)
- Конкретный товар: фикс. продажная цена 15 000 руб.

## Наценка на уровне сметы

При создании новой сметы автоматически подставляются **глобальные дефолтные наценки** (по умолчанию: материалы 30%, работы 300%). Их можно изменить на вкладке "Информация" в блоке "Наценки по умолчанию".

Изменение наценки на смете **пересчитывает все строки**, которые не имеют собственных наценок на уровне раздела или строки.

## Наценка на уровне раздела

На вкладке "Разделы" у каждого раздела есть два поля:
- **Наценка мат. %** — наценка на материалы для всего раздела
- **Наценка раб. %** — наценка на работы для всего раздела

Когда поле пустое (placeholder "По умолчанию"), раздел наследует наценку от сметы.

## Наценка на уровне строки

Для каждой строки можно задать индивидуальную наценку тремя способами:

### 1. Процент
Наценка задаётся в процентах. Пример: закупка 1 000 руб., наценка 30% → продажа 1 300 руб.

### 2. Продажная цена
Задаётся желаемая продажная цена. Пример: закупка 1 000 руб., продажная цена 1 500 руб. → наценка 50%.

### 3. Фиксированная сумма
Наценка в рублях за единицу. Пример: закупка 1 000 руб., фикс. наценка 500 руб. → продажа 1 500 руб.

Эти три способа **взаимоисключающие** — для одной строки используется только один.

## Массовая наценка

1. Отметьте нужные строки галочками в таблице
2. Нажмите кнопку **"Наценка"** на панели инструментов
3. В диалоге выберите тип наценки для материалов и/или работ
4. Нажмите "Применить"

Также можно **сбросить** индивидуальную наценку, чтобы строка снова наследовала наценку от раздела/сметы.

## Колонки в таблице

В таблице строк сметы доступны колонки:

| Колонка | Описание |
|---------|----------|
| Закупка мат. | Закупочная цена материала за единицу |
| Закупка раб. | Закупочная цена работы за единицу |
| Итого закупка мат. | Кол-во × Закупка мат. |
| Итого закупка раб. | Кол-во × Закупка раб. |
| Наценка мат. % | Эффективный процент наценки (скрыта по умолчанию) |
| Наценка раб. % | Эффективный процент наценки (скрыта по умолчанию) |
| Продажа мат. | Продажная цена материала за единицу |
| Продажа раб. | Продажная цена работы за единицу |
| Итого продажа мат. | Кол-во × Продажа мат. |
| Итого продажа раб. | Кол-во × Продажа раб. |

Видимость колонок настраивается через настройки столбцов (иконка шестерёнки).

## Вкладка "Итоги"

На вкладке "Итоги" отображаются корректные суммы:
- **Продажа**: итого по всем строкам с наценкой
- **Закупка**: итого по закупочным ценам
- **Прибыль**: Продажа − Закупка
- **Прибыль %**: (Прибыль / Продажа) × 100

## Экспорт в Excel

Доступны два режима экспорта:

- **Экспорт (внутренний)** — все колонки: закупочные цены, наценки, продажные цены. Для внутреннего использования.
- **Экспорт (для заказчика)** — только продажные цены. Закупочные цены и наценки скрыты.

## Глобальные дефолты

Глобальные дефолтные наценки (для всех новых смет) настраиваются в Django Admin → "Наценки по умолчанию". Изменение глобальных дефолтов влияет только на **новые** сметы.
