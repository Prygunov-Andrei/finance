# Сметы — Руководство пользователя

## Импорт сметы

### Импорт из Excel

1. Откройте смету и нажмите **Импорт файла**
2. Нажмите **Excel** и выберите файл (.xlsx)
3. В окне предпросмотра проверьте распознанные строки
4. При необходимости назначьте строки заголовками разделов (кнопка папки слева от строки)
5. Нажмите **Импортировать** — строки будут добавлены в смету

### Импорт из PDF (постраничный)

Для больших PDF-файлов (десятки и сотни страниц) используется постраничная обработка через ИИ. Это может занимать **10 и более минут**.

1. Откройте смету и нажмите **Импорт файла**
2. Нажмите **PDF** и выберите файл
3. Начнётся постраничная обработка — строки появляются в таблице по мере распознавания
4. Вы можете работать с уже загруженными строками, не дожидаясь окончания

> **Совет:** PDF-файлы можно импортировать и из файлов проекта (спецификаций) — обработка будет аналогичной: постраничная с прогресс-баром. Подробнее в разделе «Импорт из файлов проекта».

### Сворачивание диалога во время обработки

Если нужно заняться другими делами пока PDF обрабатывается:

1. Нажмите кнопку **Свернуть** (или нажмите Esc / кликните вне окна)
2. Диалог свернётся в компактный **синий индикатор** в правом нижнем углу экрана
3. Индикатор показывает текущий прогресс (номер страницы и процент)
4. Можно работать с другими разделами ERP — индикатор не мешает
5. Когда обработка завершится — **прозвучит звуковой сигнал** и диалог автоматически развернётся на экране предпросмотра

**Отмена импорта из свёрнутого вида:** нажмите **X** на индикаторе.

**Кнопка "Остановить"** — прекращает обработку оставшихся страниц, но уже распознанные строки сохраняются и доступны для импорта.

> **Важно:** Не уходите со страницы сметы пока идёт обработка. Переход на другую страницу ERP прервёт отслеживание прогресса (хотя обработка на сервере продолжится).

## Работа с разделами

Разделы помогают структурировать смету (например: «Вентиляция», «Электрика», «Водоснабжение»).

### Назначить строку заголовком раздела

1. Найдите строку, которая должна стать заголовком раздела
2. Нажмите кнопку с иконкой папки слева от строки
3. Строка станет заголовком раздела:
   - Выделена синим цветом и жирным шрифтом
   - Все данные кроме наименования исчезают
   - Все строки ниже до следующего заголовка автоматически принадлежат этому разделу

### Снять раздел

1. Нажмите кнопку папки на заголовке раздела (синяя строка)
2. Раздел будет снят — строка вернётся к обычному виду (с нулевыми ценами)
3. Все строки этого раздела переместятся в предыдущий раздел

## Перемещение строк

### Перемещение внутри раздела

Используйте стрелки **вверх/вниз** в колонке слева от строки:
- Нажмите **▲** — строка переместится на одну позицию вверх
- Нажмите **▼** — строка переместится на одну позицию вниз
- Перемещение мгновенное — строка двигается сразу без ожидания
- Можно нажимать стрелки быстро подряд для перемещения на несколько позиций

### Перемещение между разделами

1. Нажмите правой кнопкой мыши на строке
2. В контекстном меню выберите целевой раздел
3. Строка переместится в конец выбранного раздела

### Массовое перемещение

1. Отметьте нужные строки чекбоксами слева
2. Введите номер позиции в поле «Позиция» на панели инструментов
3. Нажмите «Перенести» — выбранные строки переместятся на указанную позицию

## Редактирование строк

- **Inline-редактирование**: кликните на ячейку, чтобы изменить значение
- **Изменение ширины колонок**: перетащите границу заголовка колонки
- **Удаление строки**: кнопка корзины справа от строки
- **Массовое удаление**: отметьте строки чекбоксами слева → нажмите кнопку «Удалить (N)»

## Объединение строк

После импорта спецификации из PDF одна позиция оборудования может оказаться разбита на несколько строк (например: «Приточная установка», «в том числе», «Комплект автоматики»). Их можно объединить:

1. Отметьте чекбоксами строки, которые нужно объединить (минимум 2)
2. Нажмите кнопку **Объединить (N)** в панели инструментов
3. Строки объединятся в одну:
   - Наименование и модель — склеиваются через пробел
   - Количество, единица измерения, цены — берутся из первой (верхней) строки
   - Нумерация строк пересчитывается автоматически

**Ограничения:**
- Все объединяемые строки должны быть в одном разделе
- Нужно выбрать минимум 2 строки

## Подбор работ (монтажных расценок)

Система автоматически подбирает расценки из прайс-листа для каждой строки сметы.

### Как запустить

1. Убедитесь что у сметы указан **прайс-лист** (вкладка "Информация")
2. В редакторе строк нажмите **"Подобрать работы"** (иконка волшебной палочки)
3. В диалоге нажмите **"Запустить подбор работ"**
4. Подбор идёт в фоне — вы можете работать с другими разделами
5. По завершении — звуковой сигнал и таблица результатов

### Результаты подбора

Каждая строка получает цветной бейдж источника:
- **Зелёный (По умолч. / История)** — надёжное совпадение (ранее подтверждённое)
- **Синий (Прайс)** — найдено в прайс-листе сметы
- **Голубой (Знания)** — найдено в базе знаний системы
- **Жёлтый (Категория)** — по категории товара
- **Оранжевый (Fuzzy)** — по похожести названий
- **Фиолетовый (LLM)** — подобрано искусственным интеллектом
- **Розовый (Web)** — найдено через поиск в интернете
- **Красный (Не найдено)** — система не нашла подходящую расценку

### Как принять/отклонить

- Отметьте галочками строки, которые хотите принять
- Кнопка **"Принять все > 70%"** автоматически отметит надёжные совпадения
- Нажмите **"Применить"** — выбранные расценки запишутся в смету
- Человеко-часы рассчитаются автоматически

### Самообучение

Система учится с каждой сметой:
- Принятые результаты повышают уверенность для будущих подборов
- Ручные коррекции — самый надёжный источник обучения
- С каждой сметой подбор становится точнее и быстрее

## Управление базой знаний

Операторы могут ускорить обучение системы тремя способами.

### 1. Редактирование .md файлов

Знания о товарах хранятся в `data/knowledge/products/` как Markdown-файлы.

**Формат файла:**
```markdown
# кондиционер настенный

## Подходящие расценки
- **C-001** Монтаж кондиционера настенного (confidence: 0.95, verified)

## Источник знаний
- LLM semantic match (2026-03-25): обоснование

## Примечания оператора
Для инверторных моделей > 7 kW использовать C-003 (усиленный монтаж).
```

После редактирования файлов выполните:
```bash
python manage.py sync_knowledge_md
```
Или подождите 30 минут — автосинхронизация.

**Примеры типовых .md файлов для ОВиК оборудования:**

Файл `data/knowledge/products/кондиционер-настенный.md`:
```markdown
# кондиционер настенный

## Подходящие расценки
- **C-001** Монтаж кондиционера настенного (confidence: 0.95, verified)

## Источник знаний
- Ручной ввод (2026-03-27): стандартная расценка для настенных сплит-систем до 7 кВт

## Примечания оператора
Для инверторных моделей мощностью > 7 kW использовать C-003 (усиленный монтаж).
Для мульти-сплит систем — отдельная расценка C-010.
```

Файл `data/knowledge/products/приточная-установка.md`:
```markdown
# приточная установка

## Подходящие расценки
- **V-001** Монтаж приточной установки (confidence: 0.95, verified)

## Источник знаний
- Ручной ввод (2026-03-27): стандартная расценка для приточных установок всех типов

## Примечания оператора
Включает: установку на виброопоры, подключение к воздуховодам и электропитанию.
Не включает: монтаж воздуховодов (V-002), пусконаладку (V-010).
```

Файл `data/knowledge/products/воздуховод-прямоугольный.md`:
```markdown
# воздуховод прямоугольный

## Подходящие расценки
- **V-002** Монтаж воздуховода прямоугольного (confidence: 0.95, verified)

## Источник знаний
- Ручной ввод (2026-03-27): единица измерения м.п., включает крепление

## Примечания оператора
Для воздуховодов сечением > 800x500 мм применять коэффициент 1.5.
Для огнезащитных воздуховодов — отдельная расценка V-015.
```

### 2. Расценка по умолчанию на товаре

При заведении товара в каталоге укажите поле **"Расценка по умолчанию"** — это мгновенная привязка с максимальной уверенностью.

### 3. Коррекция результатов подбора

При принятии/отклонении результатов подбора система автоматически запоминает решения и учитывает их при следующих подборах.
