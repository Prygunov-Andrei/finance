# Руководство директора: как выдать сотруднику доступ в ERP

После первого официального деплоя (апрель 2026) полный доступ в ERP имеют только три директора: Веракса, Савинов и Ефимова. Остальные учётные записи обнулены — пока директор не выдаст права, сотрудник разделы системы не видит.

Выдача доступа — процесс из трёх шагов:

1. Создать учётную запись (логин).
2. Задать пароль.
3. Настроить разделы ERP, в которые сотрудник может заходить.

## Шаг 1. Создать учётную запись

1. Откройте раздел **Справочники → Персонал**.
2. В списке найдите сотрудника (поиск по ФИО) или создайте нового через кнопку «Новый сотрудник».
3. Откройте карточку сотрудника — вкладка **Основное** открыта по умолчанию.
4. В блоке **Учётная запись (User)** — если в выпадающем списке стоит «Не привязана», справа от списка будет кнопка **Создать**.
5. Нажмите **Создать** — откроется окно «Создать учётную запись».
6. Логин предзаполнен транслитом ФИО (например, `Савинов Андрей → savinov.a`). При необходимости измените.
7. Нажмите **Создать** в окне.

На этом шаге User в ERP появляется, но войти в систему под ним ещё нельзя — у него нет пароля.

## Шаг 2. Задать пароль

После создания учётной записи в карточке сотрудника появится блок **Пароль доступа**.

1. Введите **Новый пароль** (минимум 8 символов, не слишком простой — Django отвергнет распространённые пароли и слишком короткие).
2. Повторите его в поле **Повторите пароль**.
3. Нажмите **Установить пароль**.
4. Появится сообщение «Пароль установлен». С этого момента сотрудник может войти в систему под выданным логином и паролем.

**Важно:** введённый пароль не отображается нигде в интерфейсе позже — передайте его сотруднику лично (мессенджер, СМС, разговор) и попросите сменить при первом входе (через раздел «Мой профиль»).

Если нужно **сбросить** пароль сотруднику — просто введите новый и нажмите «Установить пароль» ещё раз.

## Шаг 3. Настроить разделы ERP

1. В карточке сотрудника перейдите на вкладку **Доступ ERP**.
2. Для каждого раздела выберите уровень:
   - **Нет** — сотрудник не видит раздел.
   - **Чтение** — видит, но не может менять.
   - **Редакт.** — полный доступ в разделе.
3. Изменение уровня у родительского раздела (например, «Финансы») каскадом выставит такой же уровень всем подразделам — потом подразделы можно донастроить точечно.
4. Нажмите **Сохранить** внизу карточки.

### Рекомендации по ролям

| Роль | Разделы с `edit` | Остальное |
|------|-----------------|-----------|
| Снабженец | `Снабжение и Склад` целиком, `Товары и услуги → Каталог/Модерация` | `read` для `Объекты`, `Коммерческие`; остальное `none` |
| ПТО | `ПТО` целиком, `Объекты`, `Договоры → Акты/Сметы` (read) | `none` для `Финансы`, `Настройки` |
| Финансист | `Финансы` целиком, `Одобрение платежей`, `Бухгалтерия` | `read` для `Договоры`; `none` для `ПТО`, `Снабжение` |
| HR | `Справочники и настройки → Персонал` (`edit`), `Контрагенты` (`read`) | `none` для всего остального |

## Как сотрудник меняет пароль сам

Сотрудник может сменить свой пароль через раздел **Мой профиль → Изменить пароль**. Для этого от него потребуется старый (текущий) пароль — так что устанавливаемые директором пароли достаточно сделать «начальными».

## FAQ

**У сотрудника уже есть учётная запись — как ему сменить пароль?**
Выберите его в выпадающем списке «Учётная запись (User)» карточки сотрудника, блок «Пароль доступа» станет активным.

**Случайно создал User с неверным логином — как исправить?**
Логин User не переименовывается средствами UI. Попросите администратора удалить User через Django-админку и заведите заново.

**Сотрудник не может войти — что проверить?**
1. У сотрудника привязан User (в карточке в блоке «Учётная запись (User)»).
2. Пароль был установлен (после создания User поле пароля стало активным и пароль сохранён).
3. На вкладке «Доступ ERP» хотя бы один раздел с `read`/`edit` — иначе сотрудник войдёт, но интерфейс будет пустой.
