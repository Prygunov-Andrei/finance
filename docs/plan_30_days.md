# План разработки приложения на 30 дней

## Дни 1–3: Аналитика и постановка задач
- Уточнить бизнес-требования, роли пользователей и сценарии использования.
- Согласовать структуру данных таблиц и файлов, определить ответственных за обновление.
- Подготовить детальный roadmap по сущностям: Объект, Договор, Платеж, Реестр.
- Подготовить чек-лист проверок для следующих этапов.

## Дни 4–6: Архитектура и окружение
- Спроектировать архитектуру backend (Django REST) и фронтенда.
- Настроить рабочее окружение, CI/CD и базовые проверки качества кода.
- Определить формат обмена данными (Excel/CSV/JSON), регламент импорта и хранения исходников.
- **Тестирование**: smoke-тест окружения, проверка пайплайнов CI, линтеров и статического анализа.

## Дни 7–9: Модели и базы данных
- Реализовать модель `Object` с адресом и связями.
- Спроектировать и создать модель `Contract` (связь с объектами, реквизиты).
- Подготовить первичные миграции и документацию по схемам.
- **Тестирование**: unit-тесты моделей и миграций, проверка корректности ограничений и уникальности данных.

## Дни 10–12: Платежи и реестр
- Создать модели `Payment` и `PaymentRegistry` с нужными полями.
- Определить связи с договорами и учёт источников данных.
- Настроить валидацию, индексы и начальные сериализаторы.
- **Тестирование**: покрыть сериализаторы и бизнес-правила unit-тестами, выполнить тестовый импорт минимального набора данных.

## Дни 13–15: Импорт данных
- Разработать процессы загрузки таблиц (Excel/CSV) и файлов с документами.
- Реализовать парсеры и валидацию исходных данных.
- Добавить хранение ссылок на документы и логи импорта.
- **Тестирование**: интеграционные тесты процессов импорта, проверка логирования ошибок, имитация некорректных файлов.

## Дни 16–18: Бизнес-логика и расчёты
- Реализовать вычисление cash-flow по объектам и договорам.
- Добавить агрегации по периодам, статусы платежей, задолженности.
- Подготовить unit-тесты для критичных расчётов.
- **Тестирование**: сценарные тесты расчётов на наборе эталонных данных, проверка граничных случаев.

## Дни 19–21: API и документация
- Создать REST endpoints для объектов, договоров, платежей, отчётов.
- Описать спецификацию (OpenAPI/Swagger), добавить примеры запросов.
- Настроить пагинацию, фильтры, сортировки.
- **Тестирование**: API-тесты (автоматизированные и мануальные) по основным сценариям, проверка авторизации и ограничений.

## Дни 22–24: Фронтенд (базовый интерфейс)
- Реализовать базовые экраны: список объектов, детали договора, отчёты.
- Подключить визуализацию cash-flow (графики, таблицы).
- Добавить загрузку ссылок на документы и историю импортов.
- **Тестирование**: UX-/UI-проверки, кросс-браузерные smoke-тесты, ручные проверки сценариев пользователя.

## Дни 25–27: Тестирование и отладка
- Провести функциональное тестирование API и интерфейса.
- Проверить корректность процессов импорта и консистентность данных.
- Подготовить чек-лист регресса и устранить критичные дефекты.
- **Тестирование**: полная регрессия, нагрузочные тесты ключевых отчётов, анализ покрытия тестами.

## Дни 28–30: Финализация и запуск
- Обновить документацию (пользовательскую и техническую).
- Настроить продакшн-окружение, выполнить пробный деплой.
- Провести демонстрацию, собрать обратную связь, зафиксировать план улучшений.
- **Тестирование**: финальный smoke-тест на продакшн-среде, контрольный прогон чек-листа релиза.

## Внедрение на предприятии (до 15 января)
- Подготовить инструкции для пользователей и администратора данных.
- Организовать обучение ключевых сотрудников и пилотный запуск.
- Настроить расписание регулярных импортов и отчётности.
- Собрать обратную связь, запланировать итерацию по улучшениям.
