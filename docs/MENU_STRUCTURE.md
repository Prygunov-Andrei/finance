# Структура левого меню ERP-системы «Август»

> Актуальное состояние на февраль 2026. Источник истины: `frontend/components/Layout.tsx`

## Принципы навигации

- **Process-centric**: меню организовано по бизнес-процессам / отделам, а не по типам данных
- **Одно каноническое место**: редактирование сущности только в одном месте; в остальных — read-only / ссылки
- **Максимум 2 уровня** вложенности в сайдбаре; 3-й уровень — табы внутри страниц
- **Дублированные входы** (↗) — ярлыки, ведущие к каноническому месту редактирования
- **Адаптивная карточка объекта**: 4 корневых вкладки (Основное, Заказчик, Исполнители, Настройки) с под-вкладками

**Обозначения статусов:**
- ✅ — реализовано и работает
- 🚧 — заглушка (StubPage), функционал в разработке
- ↗ — ярлык (shortcut), ведёт на каноническую страницу из другого раздела

## 3. ОБЪЕКТЫ
`/objects` — ✅

Единый список с умными фильтрами: Все / Планируются / В работе / Завершённые / Приостановлены. По умолчанию — «В работе».

**Виды списка:** табличный (по умолчанию) и мозаичный (переключатель).

**Статусы объектов:** planned (Планируется), in_progress (В работе), completed (Завершён), suspended (Приостановлен).

**Карточка объекта** (`/objects/:id`) — полная переработка:
- **Шапка**: фото-аватар (загрузка), название, адрес, описание — всё inline-редактируемое. Статус и даты — с подтверждением через модальное окно. Даты создания/обновления — read-only. Кнопки «Редактировать» и «Удалить» убраны из шапки.

## Архитектура Канбан-системы

### Общий Канбан-микросервис
Единый Django-сервис (`kanban_service`, порт 8010) с ядром `kanban_core` (Board, Column, Card) и оверлеями:
- **kanban_supply** — Снабжение (`supply` board, CardType: `supply_case`)
- **kanban_object_tasks** — Задачи по объектам (`object_tasks` board, CardType: `object_task`)
- **kanban_commercial** — Коммерческий пайплайн (`commercial_pipeline` board, CardType: `commercial_case`)

### commercial_pipeline board (12 колонок)

| # | Ключ               | Название             | Вид: Маркетинг | Вид: КП |
|---|---------------------|----------------------|:-:             |:-:      |
| 1 | new_clients         | Новые клиенты        | ✅             |         |
| 2 | meeting_scheduled   | Назначена встреча    | ✅             |         |
| 3 | meeting_done        | Проведена встреча    | ✅             |         |
| 4 | new_calculation     | Новый расчет         |                | ✅      |
| 5 | in_progress         | В работе             |                | ✅      |
| 6 | invoices_requested  | Счета запрошены      |                | ✅      |
| 7 | estimate_approval   | Утверждение сметы    |                | ✅      |
| 8 | estimate_approved   | Смета утверждена     |                | ✅      |
| 9 | kp_prepared         | Подготовлено КП      |                | ✅      |
|10 | calculation_done    | Расчет подготовлен   | ✅             |         |
|11 | no_result           | Нет результата       | ✅             |         |
|12 | has_result          | Есть результат       | ✅             |         |

Механизм «тоннеля» реализован через фильтрацию: один board, два view (prop `visibleColumnKeys` в `KanbanBoardPage`).

### CommercialCase overlay
Поля: object_name, system_name, counterparty_name, linked_tkp_ids (JSON), contacts (JSON), comments.
