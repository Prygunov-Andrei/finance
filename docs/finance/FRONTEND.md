## Компоненты

### 1. FinanceDashboard

**Файл**: `FinanceDashboard.tsx` (236 строк)

Главная панель финансового раздела.

**Функционал:**
- Отображение реальных остатков на банковских счетах (данные из `GET /api/v1/accounts/`)
- Подсчёт итогового баланса по всем активным счетам
- Иконки: банковский счёт (`Landmark`) или касса (`Banknote`)
- Заглушки (StubSection) для будущих показателей:
  - Дебиторская задолженность
  - Кредиторская задолженность
  - Прибыль
  - Оборотные средства
  - Отчёты за периоды
  - Чистые активы
  - Налоги

**API-запросы:**
- `useQuery(['accounts-active'])` → `api.getAccounts({ is_active: true })`

**UI-библиотеки:** Card, Badge, Skeleton (loading state)

---

### 2. PaymentsTabPage

**Файл**: `PaymentsTabPage.tsx` (56 строк)

Контейнер с 3 горизонтальными табами. Текущий таб хранится в URL query param `?tab=`.

**Табы:**
- `invoices` — Счета на оплату (InvoicesTab)
- `registry` — Реестр оплат (PaymentRegistryTab)
- `income` — Входящие платежи (IncomingPaymentsTab)

**Управление состоянием:**
- `useSearchParams()` — read/write `?tab=` параметр
- `Tabs.onValueChange` → `setSearchParams({ tab: value }, { replace: true })`

---

### 3. InvoicesTab

**Файл**: `InvoicesTab.tsx` (247 строк)

Список счетов на оплату с фильтрами и сводкой.

**Функционал:**
- Summary-карточки: всего счетов (сумма), просроченные (количество, сумма)
- Фильтры: тип счёта (`invoice_type`), статус (`status`)
- Таблица: тип, номер, дата, контрагент, объект, сумма, статус, срок оплаты
- Визуальная индикация:
  - Просроченные → `bg-red-50/50`
  - Долговые (is_debt) → `border-l-2 border-l-amber-400`
  - Просроченная дата → красный текст с иконкой `AlertTriangle`
- Кнопка «Создать счёт» → открывает InvoiceCreateDialog
- Клик по строке → навигация на `/supply/invoices/{id}`

**API-запросы:**
- `useQuery(['invoices', queryParams])` → `api.getInvoices(queryParams)`

### 4. InvoiceCreateDialog

**Файл**: `InvoiceCreateDialog.tsx` (753 строки)

Модальный диалог для создания нового счёта. Двухшаговая форма.

#### Шаг 1: Выбор типа

5 интерактивных карточек с иконками:
- От Поставщика (`Truck`)
- По Акту (`FileCheck`)
- Хоз. деятельность (`Building2`)
- Закупка на склад (`Package`)
- Внутренний перевод (`ArrowLeftRight`)

#### Шаг 2: Форма (зависит от типа)

| Поле | supplier | act_based | household | warehouse | internal_transfer |
|------|:-------:|:--------:|:---------:|:---------:|:----------------:|
| Файл (drag & drop) | ✅ | — | — | ✅ | — |
| Объект | ✅ | ✅ | — | — | — |
| Договор | ✅ (cascade) | ✅ (cascade) | — | — | — |
| Акт | — | ✅ (cascade) | — | — | — |
| Контрагент | ✅ | — | — | ✅ | — |
| Статья расходов | — | — | ✅ | — | — |
| Со счёта / На счёт | — | — | — | — | ✅ |
| Суммы (gross/VAT/net) | ✅ | ✅ | ✅ | ✅ | ✅ |
| Срок оплаты | ✅ | ✅ | ✅ | ✅ | ✅ |
| Долговой + пояснение | ✅ | ✅ | ✅ | ✅ | ✅ |
| Пропустить распознавание | ✅ | — | — | — | — |

**Каскадные зависимости:**
- Объект → загрузка Договоров
- Договор → загрузка Актов (для act_based)
- Выбор акта → авто-заполнение суммы

**НДС-калькулятор:**
- По умолчанию: авто-расчёт 20/120 от суммы
- Кнопка «вручную» / «авто» для переключения

**Проверка сальдо:**
- Для supplier/act_based с объектом: `GET /invoices/check_balance/`
- Предупреждение жёлтой плашкой если средств недостаточно

**API-запросы:**
- accounts, legal entities, objects, contracts, counterparties, acts, expense categories (по мере необходимости, `enabled` зависит от типа и шага)
- `useMutation` → `api.createInvoice(formData)` (multipart)

### 5. PaymentRegistryTab

**Файл**: `PaymentRegistryTab.tsx` (463 строки)

Реестр оплат с workflow-управлением.

**Функционал:**
- **Мини-баннер** сверху: остатки на всех банковских счетах (горизонтальный scroll)
- **Сводка по статусам**: карточки с количеством и суммой для каждого статуса
- **Фильтр по статусу**: сегментированные кнопки (Все / В реестре / Согласовано / Отправляется / Оплачено)
- **Таблица**: номер, контрагент, объект, сумма, статус, срок оплаты, согласование, действия
- **Визуальная подсветка строк**:
  - Оплата сегодня или просрочена → `bg-red-50`
  - Оплата на этой неделе → `bg-yellow-50`
- **Действия** (для статуса `in_registry`):
  - ✅ Согласовать (approve) — зелёная кнопка
  - ❌ Отклонить (reject) — красная кнопка, открывает диалог с комментарием
  - 🕐 Перенести (reschedule) — открывает диалог с новой датой и комментарием

**API-запросы:**
- `useQuery(['invoices-registry'])` → `api.getInvoices(status)`
- `useQuery(['accounts-active'])` → `api.getAccounts()`
- `useMutation` → approve / reject / reschedule

### 6. IncomingPaymentsTab

**Файл**: `IncomingPaymentsTab.tsx` (559 строк)

Управление входящими платежами, разделённое на 2 подтаба.

#### Подтаб «Ручной ввод»

- Список поступлений с таблицей (тип, дата, контрагент, сумма, объект, описание)
- Кнопка «Создать запись» → диалог создания
- Summary: количество записей, общая сумма

**Диалог создания:**
- Тип поступления (6 вариантов)
- Для «от заказчика» (customer_act, advance, warranty_return):
  - Каскадный выбор: Объект → Договор → Акт
- Для «прочих» (supplier_return, bank_interest, other):
  - Выбор категории из внутреннего плана счетов
- Общие поля: счёт, юр. лицо, контрагент, сумма, дата, описание
- Switch: «Наличный платёж» (`is_cash`)

#### Подтаб «Из банковской выписки»

Заглушка с описанием будущего функционала:
- Автоматический импорт входящих платежей из банковской выписки
- LLM-подбор Объекта и Договора
- Корректировка оператором

#### Кассовый журнал

Показывается внизу страницы, если есть записи с `is_cash=true`.
Отдельная таблица: дата, тип, контрагент, сумма, описание.

**API-запросы:**
- `useQuery(['income-records'])` → все записи
- `useQuery(['income-records-cash'])` → `is_cash=true`
- `useMutation` → `api.createIncomeRecord(data)`

## Общие паттерны

### Форматирование

Все компоненты используют `formatCurrency()` — `Intl.NumberFormat('ru-RU', { style: 'currency', currency: 'RUB' })`.

### Состояние загрузки

`Skeleton` компонент для индикации загрузки. Количество skeleton-строк соответствует ожидаемому количеству данных.

### Пустые состояния

Каждый список имеет пустое состояние с иконкой (`FileText`, `Inbox`), заголовком и подсказкой.

### Accessibility

- `aria-label` на всех интерактивных элементах
- `tabIndex={0}` на кликабельных строках таблиц
- `onKeyDown` обработчик (Enter) для строк
- `role="button"` на кликабельных элементах

### Навигация по меню (Layout.tsx)

Раздел «Финансы» в боковом меню:
- Дашборд → `/finance/dashboard`
- Платежи → `/finance/payments`
- Инструкции → `/finance/instructions`

Раздел «Снабжение и склад»:
- Счета на оплату → `/finance/payments?tab=invoices` (ссылка для удобства)
