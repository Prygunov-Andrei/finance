# Onboarding feedback

Обратная связь от новых разработчиков по результатам первой недели. Каждый новичок добавляет раздел со своими наблюдениями.

Цель — постоянно улучшать documentation и процессы, а не ждать квартального ретроспектива.

## Формат

```markdown
## {Имя}, {дата}

### Что было понятно сразу
- ...

### Где пришлось спрашивать у buddy
- ...

### Что противоречиво / неочевидно
- ...

### Предложения по улучшению
- ...

### Какие задачи выполнил за первую неделю
- ...
```

## Текущие записи

Пока записей нет — добавятся по мере появления команды.
