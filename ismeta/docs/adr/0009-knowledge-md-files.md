# ADR-0009. ProductKnowledge в БД + зеркало в .md файлах per-workspace

- **Статус:** Accepted
- **Дата:** 2026-04-15

## Контекст

База знаний подбора (ProductKnowledge) — ключевой ценностный актив. Нужно:
1. быстро искать (fuzzy, по нормализованному имени) — задача БД;
2. ручной редактируемости — сметчик иногда правит правила, добавляет заметки.

Варианты хранения:
1. Только БД.
2. Только `.md` файлы.
3. Гибрид: БД + зеркало в `.md`.

## Решение

Гибрид: первичный источник — БД, `.md` файлы генерируются и синхронизируются автоматически. Ручная правка `.md` — поддерживается (фоновая задача подхватывает изменения).

Пути файлов — `data/knowledge/workspaces/{workspace_id}/products/{normalized_name}.md`.

## Обоснование

- **БД нужна** для fuzzy-запросов с фильтром по workspace_id.
- **`.md` нужны** для дружественной ручной правки: сметчик/админ открывает в текстовом редакторе, правит, коммитит в git (если коробка) или просто сохраняет (если SaaS и включён ручной режим).
- **Per-workspace:** в коробке файлы лежат локально, в SaaS — в S3 с префиксом `ws/{workspace_id}/`. Никогда не смешиваем данные разных workspace'ов.

## Последствия

- **Плюс:** удобство для сметчика, возможность code-review базы знаний через git (коробочный сценарий).
- **Плюс:** backup знаний легко — папки `.md` в архив.
- **Минус:** двойная запись, возможны расхождения. Решается фоновой задачей `sync_knowledge_md_task` раз в 30 минут.
- **Минус:** при конфликте (правили и БД, и файл) выигрывает последний записавший — нужно документировать в Admin Guide.
