# ADR-0006. Не кешируем WorkItem, дёргаем ERP live

- **Статус:** Accepted
- **Дата:** 2026-04-15

## Контекст

Прайс работ — 238 позиций в проде (24 секции, 5 грейдов). Для fuzzy-подбора работ можно:
1. Кешировать WorkItem локально в БД ISMeta, обновлять через webhook.
2. Дёргать ERP live при каждом запуске подбора.

## Решение

Live, без кеша.

## Обоснование

- **Объём:** 238 строк — fuzzy-запрос выполняется <50 мс даже без оптимизации. Кеш не даёт заметного выигрыша.
- **Свежесть:** новая версия прайса активируется — следующая смета сразу считает по новым ценам.
- **Упрощение:** не нужна инвалидация кеша, миграции для таблицы, обработка webhook'а `work_item.changed`.
- **Пессимистическая оценка:** ставка растёт до 5000 работ — 200 мс fuzzy — пока терпимо. До 10 000 — 400 мс, момент добавлять кеш.

## Последствия

- **Плюс:** проще код. ISMeta меньше хранит у себя.
- **Минус:** зависимость от доступности ERP во время подбора. Решается graceful degradation (см. [`specs/04-llm-agent.md`](../../specs/04-llm-agent.md) и CONCEPT.md §4.7).

## Триггер к пересмотру

**Автоматический чек в CI/мониторинге:** ежедневный Celery-task `check_workitem_cache_threshold`:
1. Запрашивает `GET /api/erp-catalog/v1/work-items/?count_only=true` для каждого активного Workspace.
2. Если `count > 1000` — создаёт алерт в `#ismeta-alerts` и флаг `workspace.settings.workitem_cache_recommended = true`.
3. Если `count > 2000` и кеша до сих пор нет — эскалация в техлид.

**Стратегия перехода (когда триггер сработал):**
- Ввод таблицы `WorkItemCache` с минимальными полями (id, article, name, unit, hours, coefficient, grade, section_code).
- Полная синхронизация при активации, затем — инвалидация по webhook `work_item.changed` (новое событие).
- TTL 60 сек как fallback на случай потери webhook'а.
- Миграция не breaking: код matching сначала проверяет кеш, при промахе — live-запрос.

**Не-триггеры (сигналы, что пора даже при малом count'е):**
- latency fuzzy-запроса P95 > 200 мс;
- ERP catalog API latency P95 > 500 мс;
- клиент требует offline-работы > 15 минут.
