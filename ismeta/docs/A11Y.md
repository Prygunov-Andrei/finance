# Accessibility Guidelines

**Версия:** 0.1. **Дата:** 2026-04-15. **Источник:** UX ревью H.

Требования доступности для ISMeta. Target: **WCAG 2.1 AA**.

## 0. Принцип

Accessibility не только для disabled пользователей. Это:
- Лучший keyboard UX (все sметчики ценят).
- Better contrast (полезно вечером, на плохих мониторах).
- Semantic markup (помогает screen readers И SEO И future AI).
- Explicit focus states (помогает при usability testing).

## 1. Scope

### 1.1 Что обязательно в MVP

- Keyboard navigation.
- ARIA labels для критичных компонентов.
- Color contrast.
- Focus management.
- Screen reader support для public portal.

### 1.2 Что в backlog

- Full screen reader support для редактора сметы.
- Voice control.
- High contrast mode.
- Magnification support.

## 2. WCAG 2.1 AA минимальные требования

### 2.1 Perceivable

- **1.1.1 Non-text Content:** все non-text имеет text alternative. Иконки с aria-label.
- **1.3.1 Info and Relationships:** semantic HTML. `<button>` для кнопок, `<a>` для ссылок.
- **1.4.3 Contrast (Minimum):** 4.5:1 для normal text, 3:1 для large (≥18px).
- **1.4.4 Resize text:** zoom до 200% без потери функциональности.
- **1.4.10 Reflow:** без horizontal scroll на viewport ≥ 320px ширины.
- **1.4.11 Non-text Contrast:** UI-элементы (кнопки, чекбоксы) — контраст 3:1.

### 2.2 Operable

- **2.1.1 Keyboard:** вся функциональность доступна с клавиатуры.
- **2.1.2 No Keyboard Trap:** можно выйти из любого компонента клавиатурой.
- **2.1.4 Character Key Shortcuts:** shortcuts можно отключить или переназначить.
- **2.4.3 Focus Order:** порядок focus логичный (по сути чтения).
- **2.4.7 Focus Visible:** focus state всегда видим.
- **2.5.5 Target Size:** touch targets ≥ 44×44px.

### 2.3 Understandable

- **3.1.1 Language of Page:** `<html lang="ru">`.
- **3.2.1 On Focus:** focus не вызывает неожиданных изменений context.
- **3.3.1 Error Identification:** errors явно указаны.
- **3.3.2 Labels or Instructions:** все form fields имеют labels.

### 2.4 Robust

- **4.1.2 Name, Role, Value:** UI-компоненты имеют corresponding ARIA attributes.
- **4.1.3 Status Messages:** changes в status (например, "Сохранено") передаются screen reader через live regions.

## 3. Component-specific guidelines

### 3.1 Tables (редактор сметы)

- `<table>` с `<caption>`, `<thead>`, `<tbody>`.
- `<th scope="col">` для headers.
- `role="grid"` для interactive tables.
- Arrow key navigation между ячейками.
- Enter — edit cell.
- Escape — cancel.
- Tab — переход к следующей cell или сохранение и переход.

### 3.2 Dialogs (модальные окна)

- `<dialog>` element (or `role="dialog"`).
- `aria-labelledby` указывает на title.
- `aria-describedby` для описания.
- Focus moves into dialog при открытии.
- Focus trapped внутри dialog.
- Escape закрывает dialog.
- Focus возвращается на триггер при закрытии.

### 3.3 Chat with LLM agent

- Messages в `<ul role="log" aria-live="polite">`.
- Новое message announces через screen reader.
- Streaming output — polite, не aggressive.
- Tool-calls — described для screen reader.

### 3.4 Loading states

- `aria-busy="true"` на элементе, который загружается.
- Screen reader announcement: «Загружается...».
- После loading: «Готово» (with `aria-live="polite"`).

### 3.5 Error states

- `role="alert"` для критичных errors.
- Error text явно привязан к field через `aria-describedby`.
- Color + text + icon (не только color).

### 3.6 Form inputs

- `<label for="...">` для каждого input.
- `required` attribute для обязательных.
- `aria-invalid="true"` при error.
- `aria-describedby` для hints.

## 4. Keyboard shortcuts

### 4.1 Global (работают везде)

- `?` — открыть keyboard shortcuts cheatsheet.
- `Ctrl/Cmd+K` — command palette (search commands).
- `Ctrl/Cmd+/` — focus search.
- `Escape` — cancel/close.

### 4.2 Editor (редактор сметы)

- `Arrow keys` — navigate cells.
- `Tab / Shift+Tab` — next/previous cell.
- `Enter` — edit cell or confirm.
- `F2` — edit current cell.
- `Delete` — clear cell.
- `Ctrl/Cmd+Z` — undo.
- `Ctrl/Cmd+Shift+Z` — redo.
- `Ctrl/Cmd+S` — save (actually autosave, but expected).
- `Ctrl/Cmd+F` — find in estimate.
- `Ctrl/Cmd+B` — bulk operations.
- `Ctrl/Cmd+1..9` — jump to section 1-9.

### 4.3 Chat

- `Enter` — send.
- `Shift+Enter` — new line.
- `Arrow Up` — edit last message.
- `Escape` — close chat.

## 5. Color palette

### 5.1 Contrast requirements

| Element | Background | Foreground | Ratio | Pass? |
|---|---|---|---|---|
| Primary text | #ffffff | #1a1a1a | 16.6:1 | ✅ |
| Secondary text | #ffffff | #6b7280 | 5.0:1 | ✅ |
| Primary button | #2563eb | #ffffff | 4.7:1 | ✅ |
| Secondary button | #e5e7eb | #1a1a1a | 12.6:1 | ✅ |
| Error text | #ffffff | #dc2626 | 5.9:1 | ✅ |
| Success text | #ffffff | #16a34a | 3.8:1 | ⚠️ (не для body text) |
| Warning text | #ffffff | #d97706 | 4.5:1 | ✅ |

### 5.2 Color-blind friendly

- Red + Green не единственный способ индикации.
- Confirmation + check icon.
- Error + X icon + red.
- Info + i icon + blue.

### 5.3 Dark mode

- Обязательно separate palette (не invert).
- Те же contrast requirements.

## 6. Typography

- **Min font size:** 14px (для body).
- **Line height:** 1.5 для body, 1.3 для headings.
- **Font weights:** 400 (normal), 500 (medium), 700 (bold). Не используем 300 (thin, плохо читается).
- **Letter spacing:** default, no changes.

## 7. Focus management

### 7.1 Visible focus

```css
*:focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
}
```

### 7.2 Skip links

В начале страницы — «Skip to main content» (видим только для keyboard users).

### 7.3 Focus after navigation

- После route change — focus на h1.
- После opening modal — focus на first interactive element.
- После closing modal — focus возвращается к триггеру.

## 8. Screen reader support

### 8.1 Test with

- **NVDA** (Windows, free).
- **VoiceOver** (macOS, встроен).
- **ORCA** (Linux).

### 8.2 ARIA landmarks

```html
<body>
  <header role="banner">...</header>
  <nav role="navigation" aria-label="Main">...</nav>
  <main role="main">...</main>
  <aside role="complementary">...</aside>
  <footer role="contentinfo">...</footer>
</body>
```

### 8.3 Live regions

- `aria-live="polite"` — для non-urgent updates (например, «Сохранено»).
- `aria-live="assertive"` — для urgent (например, errors).
- `role="alert"` — shortcut для assertive.

---

## 9. Mobile (public режим)

- Touch targets ≥ 44×44px.
- No hover-only interactions.
- Meta viewport: `width=device-width, initial-scale=1`.
- Responsive breakpoints: 320px, 768px, 1024px, 1280px.

## 10. Testing

### 10.1 Automated

```bash
# В CI
npm install -D @axe-core/react
# Prefix для каждого component test:
await axe.run();
```

### 10.2 Manual

**Checklist перед merge:**
- [ ] Keyboard only: можно пройти workflow без мыши.
- [ ] Screen reader: VoiceOver/NVDA читает разумно.
- [ ] Zoom 200%: всё читается без horizontal scroll.
- [ ] Color contrast: проверено через browser DevTools.

### 10.3 Lighthouse

- Regression check: Accessibility score > 90 в Lighthouse.
- CI job падает при < 85.

## 11. Process

### 11.1 Design

- Каждый Figma файл — с accessibility annotations.
- Contrast checker plugin активен.
- Designer проходит accessibility check перед handoff.

### 11.2 Development

- PR checklist включает accessibility.
- Automated tests обязательны.
- Manual QA с keyboard + screen reader.

### 11.3 QA

- Regular full accessibility audit раз в квартал.
- External audit раз в год (перед сертификациями).

---

## 12. Exceptions (явные отклонения)

- Редактор сметы с 4000 строк: полная screen reader support — частично (только заголовки, не каждая cell).
- Canvas-based компоненты (если появятся): описание как альтернатива.
- Complex chart dashboard: data table alternative.

Каждое exception документируется в issue с обоснованием.
