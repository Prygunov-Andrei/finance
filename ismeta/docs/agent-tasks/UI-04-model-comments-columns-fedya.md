Backend (Петя) в E15.04 будет класть:
- `EstimateItem.tech_specs.model_name` — модель/артикул (ключ уже используется).

**Schema не меняем** (миграции запрещены без отдельного решения — CLAUDE.md). Все три поля живут в `tech_specs: JSONField`.

## Задача

### 1. Столбец «Модель» (новый)

- Вставить между **«Наименование»** и **«Ед.изм.»** в таблице items.
- Источник данных: `item.tech_specs.model_name` (fallback на `""`).
- Inline-editable: изменение обновляет `tech_specs.model_name` через существующий PATCH endpoint (если есть inline-edit tech_specs — см. DEV-BACKLOG #11; если нет inline-edit, то — readonly в этой итерации, завести subtask на inline-edit).
- Ширина: ~160px (подбирать по дизайну).
- Заголовок столбца: `Модель` (короткий). При hover tooltip: `Тип, марка, обозначение`.

### 2. Столбец «Примечание» (новый)

- Вставить в **конец таблицы** (после «Подбор», перед «Действия/Ещё»).
- Источник: `item.tech_specs.comments`.
- Inline-editable (тот же механизм что модель).
- Ширина: ~200px, `white-space: pre-wrap; overflow: hidden; text-overflow: ellipsis;` (длинные — троеточие, full — tooltip или expand).
- Заголовок: `Примечание`.

### 3. Удалить дублирование model_name под name

- Сейчас model_name отображается подстрокой под именем (см. `demo-08-pdf-import-sections-sticky.png`).
- Убрать подстроку — model теперь в своём столбце.
- Если `tech_specs.model_name` пусто — в столбце `—` (em-dash).

### 4. TypeScript типы

- Обновить `ismeta/frontend/lib/api/types.ts` (или соответствующий файл с типами Estimate) — добавить опциональные поля в `tech_specs`:

```ts
export interface EstimateItemTechSpecs {
  model_name?: string;
  brand?: string;
  source_page?: number;
  comments?: string;      // NEW — из E15.04
  system?: string;        // NEW — из E15.04 (префикс типа "ПВ-ИТП")
  // ... остальные существующие поля
}
```

### 5. Тесты

- **vitest**: компонент таблицы с mock items, содержащими `tech_specs.model_name` и `tech_specs.comments` — ассертим что значения отображаются в правильных столбцах.
- **vitest**: item без `tech_specs.comments` → столбец показывает `—`.
- **vitest**: заголовки столбцов присутствуют: `Наименование`, `Модель`, `Ед.изм.`, `Кол-во`, `Цена обор.`, `Цена мат.`, `Цена работ`, `Итого`, `Подбор`, `Примечание`.
- Регрессия: существующие тесты не сломаны.

### 6. Playwright / live проверка

Загрузить golden PDF (после hotfix'а E15.03: 142 позиции) → проверить визуально:
- «Воздуховод / 150х100» — name в своём столбце, «150х100» в соседнем «Модель».
- «Дефлектор Цаги / ф355-оц-фл» — разнесены.
- Скриншот до/после в PR.
