### #45 🟡 UX — длинное наименование перетягивает ширину столбца

**Цитата PO:** «Если длинное наименование (и я его еще удлинняю), то столбец становится все шире, а остальные сужаются. Выглядит ужасно. В какой то момент должен произойти перенос текста, или скрытие части.»

**Симптом (скрин):** строка 1 «П1/В 1-Приточно - вытяжная установка L= 755/655м 3 / ч Pc=3 0 0 Па LM DUCT Q 40-20 комплектно со см. узлом, пластинчатым рекуператором и стандартным комплектом автоматики» — 180+ символов в одной строке, таблица растягивается далеко за viewport, численные столбцы сжаты.

**Корневая причина:** в `items-table.tsx` колонка «Наименование» не имеет `max-width` / `white-space: normal`. `EditableCell` рендерит одной строкой. TanStack Table auto-size по content.

**Решения (от простого к продвинутому):**
1. **Hotfix** (15 мин): CSS `max-width: 500-600px` + `white-space: normal` + `word-break: break-word` для колонки name. Текст переносится, столбец не растягивается. Остальные получают sensible ширины.
2. **Resize by drag** (~0.5 дня): TanStack Table supports `columnResizing`. Добавить resize-handle между заголовками, state в компоненте.
3. **Persist resize** (~0.5 дня): localStorage по `estimate_id` или `user_id`. Либо backend-поле user_preferences.column_widths.

**Вопросы PO (из сообщения):**
- «Насколько сложно регулировать ширину перетягиванием?» → **средне**, 0.5 дня Фёди (TanStack Table native support).
- «Фиксировать между сессиями?» → **просто**, localStorage 1-2 часа; либо backend user_preferences — 0.5 дня (migration).

**Severity:** 🟡 minor UX (не блокер, функционально работает). Но при 150+ позициях мешает читать.

**Root cause:** **R28** (новое) — колонка name без max-width и переноса.

**Решение PO:** вариант C — перенос + resize + persist.

**Статус:** ✅ закрыт. UI-08 замержен в main @ `f2b01cc` (2026-04-23).

## Параллельные заметки по парсингу (не замечания PO, я сам отметил на скрине)

Эти проблемы PO пока не упомянул, но я вижу на скрине — фиксирую в master:

- **Kerning-артефакты не зачищены** на этом прогоне: «Pc=3 0 0 Па», «755/655м 3 / ч», «П1/В 1». R24 gap-join вчера в live-curl работал, на Андреевом прогоне не сработал — LLM variance или regression. Возвращаемся если PO отметит.
- **Symbol ³ (кубический метр)** видимо не рендерится — «м 3» / «м3» вместо «м³». PyMuPDF может терять Unicode superscript в зависимости от шрифта.

Ждём PO — возможно заметит сам. Если нет — проверим после завершения захода 1.

### #46-#49 — операции с разделами сметы (sidebar)

> «В списке разделов слева хорошо бы иметь около каждого раздела счётчик строк раздела.»

> «Возможность как добавить раздел так и объединить разделы.»

- **#48** Объединение разделов — checkboxes в sidebar + bulk toolbar + merge dialog.
- **#49** «Добавить раздел» — проверить что существующая кнопка работает.

**Severity:** 🟡 UX major (упрощает ручную коррекцию после PDF-import).

**Решение:** **UI-09** (ТЗ в `agent-tasks/UI-09-sections-move-merge-fedya.md`), 1 день, Федя. Worktree создан от `f2b01cc`.

**Статус:** ✅ закрыт. UI-09 замержен в main @ `aedf80b` (2026-04-23).
