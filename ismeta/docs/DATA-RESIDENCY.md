# Data residency

**Версия:** 0.1. **Дата:** 2026-04-15. **Источник:** архитектурное ревью E2.

Карта: какие данные, где физически живут, в каких копиях. Основа для compliance (152-ФЗ, GDPR), ответов аудиторам и корпоративным клиентам.

## 1. Классификация данных

| Класс | Примеры в ISMeta | Чувствительность |
|---|---|---|
| **ПДн** | Email посетителей публичного режима, user_id сметчиков | Высокая (152-ФЗ, GDPR) |
| **Коммерческая тайна** | Закупочные цены, маржа, ProductKnowledge, версии смет до публикации | Высокая |
| **Служебные** | AuditLog, LLMUsage, MetricAggregate | Средняя |
| **Публичные** | РРЦ в публичном режиме | Низкая |

## 2. Места физического хранения

### 2.1 Наша инсталляция (этап 1 MVP)

| Компонент | Физ. местоположение | Обоснование |
|---|---|---|
| БД ISMeta PostgreSQL | сервер 216.57.110.41 (РФ, наш) | там же, где БД ERP; 152-ФЗ OK |
| Файлы (uploads, экспорты) | тот же сервер, `/var/ismeta/uploads/` | |
| Redis | тот же сервер | |
| Backup'ы | S3 Yandex Cloud (РФ) | 152-ФЗ OK |
| Mirror в холодное хранилище | Yandex Object Storage Cold (РФ) | |

### 2.2 LLM-провайдер (по умолчанию OpenAI)

- Текст/PDF отправляется в OpenAI (инфраструктура США).
- OpenAI хранит запросы ≤ 30 дней для abuse detection (их DPA), далее удаляет.
- **Это нарушение 152-ФЗ для данных российских граждан без их явного согласия.**

### 2.3 Коробочная инсталляция (этап 3+)

- Всё развёртывается на серверах клиента (on-premise).
- Только LLM-вызовы идут наружу (если не локальный Ollama).
- Клиент сам отвечает за compliance с законами своей юрисдикции.

## 3. Картина по классам данных

### 3.1 ПДн посетителей публичного режима

**Что хранится:** email, phone (при запросе звонка), содержимое спецификации (может содержать ПДн заказчика).

**Где:** БД ISMeta РФ + LLM-провайдер (в зависимости от выбранного).

**Срок:** 30 дней после создания заявки, далее удаление.

**152-ФЗ compliance:**
- Согласие собирается на форме email.
- Право на удаление: endpoint `DELETE /api/public/v1/me` (создаётся в этапе 2).
- Трансграничная передача (OpenAI → США): **требует отдельного согласия** с указанием страны.

**Рекомендация:** для публичного режима — LLM-провайдер РФ (GigaChat / YandexGPT) по умолчанию.

### 3.2 Коммерческая тайна (ProductKnowledge, цены, маржа)

**Что хранится:** продажные цены (видят клиенты), закупочные (только мы), ProductKnowledge (наш «мозг»).

**Где:** БД ISMeta РФ.

**При LLM-вызове:** закупочные цены не отправляются в LLM (см. [`specs/04-llm-agent.md §7`](../specs/04-llm-agent.md)). ProductKnowledge — отправляется частично (для контекста агента).

**Защита:**
- шифрование at-rest (Postgres TDE / disk encryption);
- шифрование in-transit (TLS 1.3);
- access по workspace_id + role.

### 3.3 AuditLog

**Что хранится:** кто-когда-что менял в смете.

**Где:** БД ISMeta РФ.

**Срок:** 1 год (5 лет для `instance_admin` и `transmit/restore`).

**В LLM:** никогда не отправляется.

## 4. Что уезжает в облачный LLM

| Данные | Отправляется? | Когда |
|---|---|---|
| Наименование позиции | ✅ | Recognition, Tier 6/7, agent |
| Модель/бренд | ✅ | Recognition, Tier 6/7, agent |
| ТТХ | ✅ | Recognition, Tier 6/7, agent |
| Quantity, unit | ✅ | Recognition, agent |
| Наши закупочные цены | ❌ | Никогда |
| Наши продажные цены | ✅ (частично) | При запросе сравнений у агента |
| ProductKnowledge паттерны | ✅ | Agent's `search_knowledge` |
| Контрагенты (ФИО, email) | ❌ | Не передаём в LLM |
| Объекты (адреса) | ❌ | Не передаём в LLM |
| Юр. лица, реквизиты | ❌ | Не передаём в LLM |

**Правило:** в LLM уходит минимум, необходимый для задачи. Контакты и финансы — никогда.

## 5. Политика для российских клиентов

### 5.1 Обязательные требования (152-ФЗ)

1. Персональные данные граждан РФ — только на серверах в РФ.
2. Согласие на обработку — перед любой записью ПДн.
3. При трансграничной передаче — отдельное согласие с указанием страны и целей.
4. Право на удаление, изменение, выгрузку.

### 5.2 Наша реализация (к этапу 2)

- `Workspace.settings.data_residency = 'RU'` — режим для РФ-клиентов.
- В этом режиме default-LLM — GigaChat или YandexGPT.
- Если администратор workspace включает OpenAI — явное предупреждение «данные будут передаваться в США, нужно согласие пользователей».
- Форма согласия генерируется автоматически с учётом выбранного провайдера.

## 6. Политика для коробочных клиентов

Клиент получает:
- docker-compose с дефолтными настройками «всё локально, LLM — ваш выбор»;
- чек-лист compliance в Admin Guide;
- рекомендации по выбору LLM-провайдера под юрисдикцию.

Мы как вендор не отвечаем за compliance клиента, но обязательно даём инструменты.

## 7. Ответы на типовые вопросы клиентов/аудиторов

### «Где хранятся наши данные?»

Ответ: «БД ISMeta на сервере 216.57.110.41 (РФ). Бэкапы — Yandex Cloud Object Storage (РФ). LLM-провайдер — настраивается workspace'ом; по умолчанию для РФ-клиентов — GigaChat (РФ). Данные в LLM уходят в минимальном объёме — см. таблицу в `DATA-RESIDENCY.md §4`.»

### «Можно ли получить нашу копию данных?»

Ответ: «Да. Endpoint `GET /api/v1/workspaces/me/export` (создаётся в этапе 3) возвращает JSON со всеми данными workspace'а. Для форсированной выгрузки — обратитесь к support.»

### «Что будет, если я решу уйти от вас?»

Ответ: «Экспорт в стандартных форматах (JSON, XLSX) + PDF всех смет + копия `.md`-файлов базы знаний. Данные удаляются с наших серверов в течение 30 дней по запросу.»

### «Соответствуете ли вы 152-ФЗ?»

Ответ (честный): «Инфраструктура — да, хранение в РФ. LLM-провайдер зависит от настройки workspace'а; рекомендуем для РФ-клиентов использовать GigaChat/YandexGPT. Формальной сертификации 152-ФЗ у нас нет (планируется к этапу 4, SaaS-запуск).»
