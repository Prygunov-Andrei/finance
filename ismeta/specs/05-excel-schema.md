# 05. Формат .xlsx для двустороннего цикла

**Версия:** 0.1. **Назначение:** структура файла Excel при экспорте и требования к импорту.

**Рабочий пример:** [`../docs/samples/sample-estimate-data.json`](../docs/samples/sample-estimate-data.json) + скрипт [`generate-sample-xlsx.py`](../docs/samples/generate-sample-xlsx.py). Запусти `python generate-sample-xlsx.py` — получишь настоящий `sample-estimate.xlsx`, совпадающий с этой спецификацией.

## 1. Общее

Цель Excel-цикла — позволить сметчику уйти в привычную среду Excel для массовой правки, а потом вернуть изменения в ISMeta с прозрачным diff-предпросмотром.

Ключевые принципы:
- **Никаких формул пользователя не сохраняем как источник истины.** Excel считает сам, но при импорте ISMeta берёт только значения.
- **Идентичность строки определяется в первую очередь по `row_id`**, во вторую — по контент-хэшу (fallback, если `row_id` потерян).
- **Экспорт самодокументируемый**: первая книга — данные, вторая — метаданные, третья — readme-инструкция.

## 2. Структура книги

Книга содержит три листа:

| Лист | Название | Назначение | Доступ пользователя |
|---|---|---|---|
| 1 | `Смета` | Основной лист с редактируемыми данными | чтение/запись |
| 2 | `_ismeta_meta` | Метаданные (workspace, estimate_id, version, timestamps, schema_version) | скрытый (sheet hidden) |
| 3 | `Инструкция` | Readme для сметчика | только чтение, protected |

## 3. Лист «Смета»

### 3.1 Столбцы (пример для ОВиК/СС)

| Столбец | Заголовок | Тип | Обязательный | Комментарий |
|---|---|---|---|---|
| A | `row_id` | UUID string | да | скрытый (hidden column), sheet-protected от случайного удаления |
| B | `row_hash` | hex | да | скрытый |
| C | `№` | int | нет | автонумерация разделов/строк |
| D | `Раздел` | string | да для строк-заголовков | либо название раздела, либо пусто |
| E | `Подраздел` | string | нет | в MVP обычно пусто |
| F | `Наименование` | string | да | |
| G | `Модель/Артикул` | string | нет | |
| H | `Бренд` | string | нет | |
| I | `Ед.` | string | да | шт, м.п., м², м³, компл |
| J | `Количество` | decimal(18,4) | да | |
| K | `Цена закуп. материал` | decimal(18,4) | нет | |
| L | `Цена закуп. работа` | decimal(18,4) | нет | |
| M | `Наценка материал, %` | decimal(10,2) | нет | переопределение на строке |
| N | `Наценка работа, %` | decimal(10,2) | нет | |
| O | `Продажная материал` | decimal(18,4) | формула | `=K*(1+M/100)`; не читается при импорте |
| P | `Продажная работа` | decimal(18,4) | формула | `=L*(1+N/100)`; не читается при импорте |
| Q | `Итого материал` | decimal(18,4) | формула | `=J*O` |
| R | `Итого работа` | decimal(18,4) | формула | `=J*P` |
| S | `Итого` | decimal(18,4) | формула | `=Q+R` |
| T | `Примечание` | string | нет | для произвольных заметок, не используется в логике |

Столбцы `A` и `B` (row_id и row_hash) защищены sheet protection'ом и скрыты. Удаление столбца сметчиком провоцирует fallback-режим импорта.

### 3.2 Строки

Строка 1 — заголовки (frozen).
Строка 2+ — данные.

Типы строк:
- **Строка раздела**: заполнен только столбец D (`Раздел`), остальные пустые. Жирный шрифт, заливка.
- **Строка подраздела** (опционально): заполнены D и E. Чуть менее жирный стиль.
- **Строка позиции**: заполнены F-J (минимально), опционально остальные.
- **Строка итогов**: в конце листа, с формулами по столбцам Q-S (для восприятия сметчиком; при импорте игнорируется).

### 3.3 Стили

- Ширины столбцов: row_id/hash — 0 (скрытые), Номер — 5, Раздел — 30, Подраздел — 25, Наименование — 50, Модель — 20, Бренд — 15, Ед — 8, Количество/Цены — 15, Наценки — 12, Итоги — 18, Примечание — 30.
- Формат чисел: `# ##0.00` для денежных, `# ##0.####` для количеств.
- Валюта не пишется в ячейки, указана в `_ismeta_meta`.

## 4. Лист `_ismeta_meta`

Скрытый, protected. Формат key-value (столбец A — ключ, столбец B — значение).

| Ключ | Значение (пример) |
|---|---|
| schema_version | `v1` |
| workspace_id | `a1b2...` |
| estimate_id | `4f3a...` |
| estimate_version_id | `4f3a...` (совпадает с estimate_id в текущей версии) |
| estimate_number | `СМ-2026-001` |
| estimate_name | `Вентиляция корпус А` |
| currency | `RUB` |
| usd_rate | `92.5` |
| exported_at | `2026-04-15T10:23:45Z` |
| exported_by | `user-id` |
| source_hash | `sha256:...` — hash содержимого листа на момент экспорта |
| row_count | `127` |

При импорте ISMeta сверяет `schema_version`, `workspace_id`, `estimate_id`. Если не совпадает — отказ с пояснением.

## 5. Лист «Инструкция»

Только чтение (workbook protected, sheet protected). Содержит:
- короткое описание цикла (push → edit → pull);
- что можно править (наименование, количество, цены, наценки, примечания, добавлять/удалять строки);
- что **нельзя** (трогать скрытые столбцы A и B, менять метаданные);
- как сохранять файл (только .xlsx, не .csv, не .xls);
- что делать, если Excel повредил скрытые столбцы (вернуться в UI и импортировать заново).

## 6. Импорт: алгоритм на стороне ISMeta

Импорт — многошаговый процесс с предпросмотром, не моментальная замена.

### 6.1 Шаг 1: валидация книги

1. Файл — валидный .xlsx (проверяется openpyxl).
2. Лист `_ismeta_meta` присутствует, `workspace_id`/`estimate_id`/`schema_version` совпадают с ожидаемыми.
3. Лист `Смета` имеет обязательные столбцы (по заголовкам строки 1).
4. Антивирус прошёл (см. `12-security.md`).

### 6.2 Шаг 2: классификация строк

Для каждой строки `Смета` определяется:
- **keep** — row_id есть и существует в БД, содержание изменилось.
- **unchanged** — row_id есть, row_hash совпадает — никаких действий.
- **conflict** — row_id есть, row_hash в БД отличается от row_hash в файле, И в БД отличается от текущего состояния (строка правилась после экспорта). Three-way merge.
- **new** — row_id пустой (новая строка, добавленная сметчиком в Excel). Создаётся с новым UUID.
- **deleted** — row_id есть в БД, но строка отсутствует в файле (сметчик удалил строку). Soft delete (поле `is_deleted = true`).
- **malformed** — row_id невалидный UUID или отсутствует в БД → fallback-match по name+model+quantity.

### 6.3 Шаг 3: fallback-match (при потере row_id)

Если ISMeta детектирует, что в строке файла нет `row_id` (пустая ячейка или искажённое значение), включается fallback:

1. ISMeta пытается найти строку в БД по нормализованному `(name, model_name, unit, quantity)`.
2. Если найдена ровно одна — предлагается привязка к этому row_id, сметчик подтверждает.
3. Если найдено несколько кандидатов или ни одного — строка помечается «требует ручного разрешения», сметчик решает (принять как новую, привязать к конкретной, отказаться).

### 6.4 Шаг 4: diff-превью

ISMeta формирует `ImportSession` со структурой:
```json
{
  "to_update": [{"item_id":"...","changes":{"quantity":{"old":5,"new":7}}}],
  "to_create": [{"name":"...","section_id":"...","data":{...}}],
  "to_delete": [{"item_id":"...","reason":"missing_in_file"}],
  "conflicts": [{"item_id":"...","db_state":{...},"file_state":{...},"baseline_state":{...}}],
  "warnings": []
}
```

### 6.5 Шаг 5: применение

Сметчик в UI:
- просматривает каждый блок изменений,
- галочками выбирает применять/не применять,
- нажимает «Применить».

ISMeta применяет выбранное в одной транзакции, пишет `AuditLog` с source=`excel_import`.

## 7. Формулы Excel

- Формулы в столбцах O-S служат сметчику для наглядности и пересчитываются Excel'ем на лету.
- При импорте **формулы не читаются**. ISMeta пересчитывает продажные цены из своих правил (`markup_service`).
- Если сметчик поставит формулу в «количество» или «цену закуп.», Excel отдаст вычисленное значение — ISMeta возьмёт его как число.

## 8. Ограничения

- Максимальное число строк в одной смете — 15 000 (защита от случайных мега-файлов).
- Максимальный размер файла — 50 МБ.
- Поддерживаются только `.xlsx` (без `.xls`, `.csv`, `.ods`).
- UTF-8 только.

## 9. Версионирование формата

- `schema_version: v1` — текущий.
- При breaking change (удалён обязательный столбец, переименован ключ в meta) — bump до `v2`. Импорт v1 в систему с v2 — либо миграция, либо отказ с инструкцией повторного экспорта.

## 10. Тесты

- **Unit**: парсер excel против фикстур в `tests/fixtures/excel/`:
  - `happy_path.xlsx` — чистый экспорт/импорт без изменений.
  - `with_edits.xlsx` — правки quantity и цен.
  - `with_new_row.xlsx` — добавлена строка.
  - `with_deleted_row.xlsx` — удалена строка.
  - `with_moved_row.xlsx` — строка переставлена в другой раздел (не поддержано в MVP — выдаём warning).
  - `corrupted_row_id.xlsx` — ручное удаление скрытых столбцов.
  - `wrong_schema.xlsx` — несовпадение workspace/estimate_id.
  - `with_formulas.xlsx` — формулы Excel вместо чисел.
- **Integration**: e2e round-trip (экспорт → модификация через openpyxl → импорт → проверка).
