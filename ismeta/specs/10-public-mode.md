# 10. Публичный режим ISMeta

**Версия:** 0.1. **Назначение:** UX, API, технические требования публичной части. Добавляется на этапе 2 (не MVP).

## 1. Роль публичного режима

Посетитель сайта Августа загружает PDF/Excel со спецификацией и получает предварительный расчёт сметы с настраиваемой ставкой часа. Цели:
- собрать квалифицированные лиды для отдела продаж;
- продемонстрировать возможности подбора как маркетинговый инструмент;
- не раскрывать закупочные цены и маржу.

Не цели:
- заменить работу сметчика (после демонстрации заявка уходит к нему);
- работать со сметами в 10 000 строк публично (rate-limit, максимум 500 строк на одну публичную смету).

## 2. URL и развёртывание

- Публичный URL: `https://smeta.august.ru/` (или встраивание в публичный сайт iframe'ом).
- Отдельный фронтенд-бандл, оптимизированный для первого экрана (SSR Next.js).
- Отдельный воркспейс `public` (workspace_id, выделенный под публичные сметы; НЕ тот же, что у сметчиков).
- На одной ISMeta-инсталляции: один публичный воркспейс, много пользовательских.

## 3. Жизненный цикл публичной сметы

```
[Посетитель]
   │
   │ заходит на smeta.august.ru
   ▼
[Экран 1: Лендинг + форма email]
   │
   │ вводит email → запрос OTP
   │ OTP приходит за 5-30 секунд
   ▼
[Экран 2: Ввод OTP]
   │
   │ подтверждает → получает session_token
   ▼
[Экран 3: Загрузка файла]
   │
   │ загружает PDF/Excel (до 20 МБ, до 100 страниц)
   ▼
[Экран 4: Статус обработки]
   │
   │ polling каждые 5 сек
   │ этапы: "Распознаём файл" → "Создаём позиции" →
   │        "Подбираем цены" → "Формируем смету"
   ▼
[Экран 5: Просмотр сметы]
   │
   │ разделы/строки, материалы в РРЦ, работы по часам
   │ виджет «Стоимость часа по грейду» с моментальным пересчётом
   │ кнопки «Скачать PDF», «Заказать звонок»
   ▼
[Опционально: экран 6 — заказ звонка]
   │
   │ имя, телефон, удобное время
   ▼
[Заявка → ERP через webhook → смета появляется у сметчика]
```

## 4. Ограничения и rate limits

| Параметр | Значение |
|---|---|
| Макс. файлов на одну заявку | 5 |
| Макс. страниц на файл | 100 |
| Макс. размер файла | 20 МБ |
| Макс. строк в публичной смете | 500 |
| Rate limit: заявок на email | 5 в сутки |
| Rate limit: заявок на IP | 10 в сутки |
| Rate limit: OTP-запросов на email | 5 в сутки |
| TTL access_token | 30 минут |
| TTL публичной сметы (до удаления) | 30 дней |

## 5. API публичного режима

Все endpoints — в `ismeta/backend/public/`, URL-префикс `/api/public/v1/`.

Auth:
- unauthenticated (CAPTCHA-защита или Cloudflare Turnstile): `POST /auth/request-otp`, `POST /auth/confirm-otp`;
- authenticated via session_token: остальные.

### 5.1 Auth

- `POST /api/public/v1/auth/request-otp` → `{email, captcha_token}` → `{sent_at}`. Rate-limited.
- `POST /api/public/v1/auth/confirm-otp` → `{email, otp}` → `{session_token, expires_at}`.
- `POST /api/public/v1/auth/refresh` — не поддерживается (сессия короткая).

### 5.2 Заявка на расчёт

- `POST /api/public/v1/requests/` (multipart с файлами) → `{request_id, status:"uploaded"}`.
- `GET /api/public/v1/requests/{request_id}` → `{status, progress, estimated_completion_at, result_url?}`.
- `POST /api/public/v1/requests/{request_id}/cancel` → отмена.

### 5.3 Просмотр готовой сметы

- `GET /api/public/v1/estimates/{request_id}/` → смета с публичными полями.
- `GET /api/public/v1/estimates/{request_id}/sections/`.
- `GET /api/public/v1/estimates/{request_id}/items/`.
- `POST /api/public/v1/estimates/{request_id}/recalculate` → `{hourly_rates: {grade_1:..., grade_5:...}}` → пересчитанные работы (только в рамках сессии, не сохраняется на сервере).
- `GET /api/public/v1/estimates/{request_id}/export/pdf` → PDF.

### 5.4 Обратный звонок

- `POST /api/public/v1/callbacks/` → `{request_id, name, phone, preferred_time, comment?}` → создаёт заявку и push-событие в ERP.

## 6. Поля публичной сметы

Посетитель видит по каждой строке:
- наименование;
- модель/артикул (если есть);
- бренд (если есть);
- единицу измерения;
- количество;
- цену материала — **РРЦ** (через `PublicPricingConfig` из ERP, см. CONCEPT §5.3);
- человеко-часы работы (из WorkItem.hours × quantity);
- грейд работы (для расчёта стоимости);
- стоимость работы (hours × грейд.ставка_посетителя);
- сумму по строке.

Посетитель НЕ видит:
- закупочные цены;
- поставщиков;
- маржу;
- обоснование подбора;
- confidence;
- `match_source`;
- ТТХ, если они не в `tech_specs` как finalized.

## 7. Виджет «Стоимость часа по грейду»

### 7.1 Структура

Статично справа от таблицы позиций. Показывает 5 полей ввода:

| Грейд | Описание | Поле ввода | Дефолт (₽/час) |
|---|---|---|---|
| 1 | Монтажник 1 разряда | `input[type=number]` | 600 |
| 2 | Монтажник 2 разряда | `input[type=number]` | 780 |
| 3 | Монтажник 3 разряда | `input[type=number]` | 960 |
| 4 | Монтажник 4 разряда | `input[type=number]` | 1140 |
| 5 | Монтажник 5 разряда | `input[type=number]` | 1320 |

Дефолтные значения = ставка ERP × наценка (из `PublicPricingConfig.work_markup`). На проде ставка грейда 3 — 800 ₽, с наценкой 20% для публичного = 960 ₽.

### 7.2 Поведение

- При изменении любого поля через `onBlur` или `debounce(300ms)` — пересчитываются все строки работ на фронте (JS).
- В UI меняются: стоимость строки работы, итог раздела по работам, общий итог.
- Отображается подпись «ставки можно изменить, материалы не меняются».
- Сервер не вызывается для пересчёта — всё локально (данные уже пришли с hours + quantity на строке, множитель — только ставка).

### 7.3 Валидация

- Диапазон 0-100000 ₽/час; вне — блок с предупреждением.
- Только числа; буквы — блок.

## 8. Публичный PDF

- Шаблон — простой табличный (единый для публичного режима, без кастомизации логотипов).
- Содержит:
  - заголовок: «Предварительный расчёт от Август Климат»;
  - дату расчёта, номер заявки;
  - контакты Августа (из настроек PublicPortalConfig);
  - таблицу позиций с выбранными ставками грейдов;
  - итоги;
  - дисклеймер: «Расчёт предварительный, не является коммерческим предложением».

## 9. Защита

### 9.1 Anti-abuse

- Cloudflare Turnstile / reCAPTCHA на запросе OTP и загрузке файла.
- Анти-DDoS через edge (Cloudflare или встроенный у хостинга).
- Запрещён hot-link на `/download/` — только через session_token.

### 9.2 Безопасность файлов

- AV-сканирование перед обработкой.
- Sandbox-директория с TTL.
- Отказ файлов не .pdf / .xlsx / .xls.

### 9.3 Изоляция публичного workspace

- Публичные сметы в workspace `public`, НЕ в сметчиковском.
- LLM-провайдер публичного workspace — отдельный (может быть более дешёвый).
- База знаний публичного workspace — копия снимка со сметчикового (обновляется раз в сутки).

## 10. UX-требования

- Загрузка экрана 1 (лендинг) — не более 1.5 сек (LCP).
- Экран статуса: прогресс-бар с живой анимацией (а не просто процент).
- Мобильная адаптация для экранов 1-4 (просмотр сметы — рекомендуется desktop).
- Ошибки — понятные, на русском, с next-step.

## 11. Push заявки в ERP

При заказе звонка или при успешном завершении сметы — ISMeta отправляет webhook в ERP:

```
POST /api/erp/v1/ismeta/public-leads
{
  "ismeta_request_id": "uuid",
  "email": "...",
  "name": "...",
  "phone": "...",
  "preferred_time": "...",
  "estimate_summary": {
    "total_items": 127,
    "total_sale_materials": 1200000,
    "total_sale_works": 350000
  },
  "ismeta_estimate_url": "https://admin.smeta.august.ru/..."
}
```

ERP создаёт карточку лида, сметчик видит её в списке. Для сметчика смета появляется в ISMeta в workspace ERP-Августа (копия публичной, уже не read-only).

## 12. Acceptance (из `07-mvp-acceptance.md §4`)

- Посетитель проходит весь цикл от email до PDF-скачивания.
- Rate limits работают (перебор email'ов даёт 429).
- CAPTCHA защищает от ботов (ручной тест с curl без captcha_token — 400).
- JS-пересчёт срабатывает менее чем за 300 мс на смете в 500 строк.
- Публичный workspace изолирован: приватные сметы сметчиков не видны через `/api/public/v1/`.
- Заявка на звонок приходит в ERP за ≤ 5 секунд.

## 13. Метрики публичного режима

Фиксируем в `11-metrics.md`:
- конверсия «загрузил → получил смету»;
- конверсия «получил смету → оставил заявку»;
- количество публичных смет в день / месяц;
- средний размер публичной сметы (строк);
- время обработки (P50, P95);
- доля 4xx/5xx ошибок;
- расход LLM-токенов на публичную смету.
